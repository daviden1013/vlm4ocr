    :sparkles:Overview

    vlm4ocr provides a simple way to perform OCR using the power of modern Vision Language Models (VLMs). A drag-and-drop web application is included for easy access. The Python package supports concurrent batch processing for large amount of documents. CLI provides lightweight access to most OCR features without the burden of coding.

    :zap:Feature Highlights

    • Extensive support for popular models and platforms

      We provide built-in support for OpenAI-compatible servers (e.g., vLLM, llama.cpp), Ollama, OpenAI API, and Azure. All open-weights VLMs and OpenAI models are supported. Latest VLMs including Qwen2.5-VL, Llama-3.2, LLaVa-1.5 are available.

    • High throughput batch processing

      Concurrently process multiple images/pages with asynchronous inferencing. Balance performance and cost by setting concurrent_batch_size.

    • Extensive support for input and output formats

      Multiple image and file formats are supported:
    • PDF
    • TIFF
    • image (PNG, JPG/JPEG, BMP, GIF, WEBP)

    Output formats: - plain text - Markdown - HTML - ### Multiple flatforms: :snake:Python package, :earth_americas:Web App, and :computer:CLI #### Web App A Flask web App can be hosted locally for drag-and-drop access to OCR.

    https://github.com/user-attachments/assets/b196453c-fd2c-491a-ba1e-0a77cf7f5941 #### Python package We provide Python classes VLMEngine's to configure your preferred VLM inference engines hosted locally or remotely, and an OCREngine class with all functionality. 

    from vlm4ocr import OpenAIVLMEngine, OllamaVLMEngine, AzureOpenAIVLMEngine, OCREngine

vlm_engine = OpenAIVLMEngine(model="Qwen/Qwen2.5-VL-7B-Instruct", base_url="http://localhost:8000/v1", api_key="EMPTY")
ocr = OCREngine(vlm_engine, output_mode="markdown")

# Batch OCR for multiple image/pdf files
ocr_results = ocr.run_ocr([image_path, pdf_path], concurrent=True, concurrent_batch_size=32)
    #### CLI Command line interface (CLI) tool is installed automatically with the Python package, providing a lightweight access to most OCR features. 
    # OpenAI compatible API (batch processing)
vlm4ocr --input_path /examples/synthesized_data/ \
        --output_mode markdown \
        --vlm_engine openai_compatible \
        --model Qwen/Qwen2.5-VL-7B-Instruct \
        --api_key EMPTY \
        --base_url http://localhost:8000/v1 \
        --concurrent \
        --concurrent_batch_size 4 \

    :vertical_traffic_light:Prerequisites

    • Python 3.x
    • For PDF processing: poppler library.
    • At least one VLM inference engine setup (Ollama, OpenAI/Azure API keys, or an OpenAI-compatible API endpoint).
    pip install ollama # For Ollama
pip install openai # For OpenAI (compatible) and Azure OpenAI

    :cd: Installation

    The Python package and CLI can be installed via 

    pip install vlm4ocr
    The web App is available on :whale:Docker Hub 
    docker pull daviden1013/vlm4ocr-app:latest
docker run -p 5000:5000 daviden1013/vlm4ocr-app:latest
    Open your web browser and navigate to: http://localhost:5000
